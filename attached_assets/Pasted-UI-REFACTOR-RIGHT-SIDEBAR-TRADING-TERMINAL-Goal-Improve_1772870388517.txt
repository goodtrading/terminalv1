UI REFACTOR — RIGHT SIDEBAR (TRADING TERMINAL)

Goal:
Improve readability and usability of the right sidebar for beginner and intermediate traders without changing any backend logic.

Important:
Do NOT modify backend logic, data models, or API calls.
Only refactor the UI presentation.

Current panels in the sidebar:

1. Trade Setup Quality
2. Execution State
3. Daily Scenarios

We will improve hierarchy, spacing, and clarity.

------------------------------------------------

STEP 1 — MAKE EXECUTION STATE THE PRIMARY PANEL

Execution State represents the actual trading decision and must be visually dominant.

Changes:

• Increase font size of STATUS value
• Increase spacing between rows
• Make values visually stronger than labels

Target layout:

EXECUTION STATE

STATUS        EXECUTE
DIRECTION     SHORT
RISK          MEDIUM

Implementation guidelines:

- Labels: small uppercase text
- Values: larger font-weight (bold)
- Add spacing between rows (8–12px)

------------------------------------------------

Use clear color semantics for beginners.

Rules:

STATUS

AVOID ENTRY → red
WAIT → yellow

DIRECTION

SHORT → red

RISK
LOW → green

HIGH → red

Apply color only to the value, not the label.
