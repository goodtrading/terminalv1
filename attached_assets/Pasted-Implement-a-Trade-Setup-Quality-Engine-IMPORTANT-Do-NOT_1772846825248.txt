====================================================
LOGIC MODEL
====================================================

Build a scoring model using the current data.

1. Scenario confidence
- high probability scenario adds score
- medium probability adds smaller score
- low probability reduces score

2. Gamma structure
- LONG GAMMA + positive gammaPressure + high gammaConcentration = cleaner/stabler setup
- SHORT GAMMA + unstable structure = lower quality unless momentum conditions align

3. Dealer hedging flow
- BUYING or SELLING with MEDIUM/HIGH coherence adds score if aligned with active scenario
- conflicting hedge flow reduces score

4. Confirmation toggles
Use the existing UI toggles as live confirmations:
- absorption at magnet
- bid holding
- delta divergence
- OI stable

More confirmations = higher setup quality.

Suggested weighting:
- 3 or 4 confirmations active = strong boost
- 2 confirmations active = moderate boost
- 0 or 1 = weak

5. Volatility risk
Use:
- gamma regime
- accelerationRisk
- scenario type

Rules:
- LONG GAMMA + LOW acceleration risk = LOW vol risk
- SHORT GAMMA or VOL scenario = HIGH vol risk
- otherwise MEDIUM

6. Flow state
Map from current structure:
- STABLE if long gamma + positive gamma pressure + high concentration
- SUPPORTIVE if hedge flow bias supports current scenario
- SUPPRESSIVE if flow works against the scenario
- EXPANSIVE if short gamma / volatility acceleration conditions dominate

7. Final quality mapping
Example:
score 8+ = A
score 6-7 = B
score 4-5 = C
below 4 = D

8. Condition mapping
- CONFIRMED if active scenario exists and confirmations align
- DEVELOPING if some confirmations align
- WEAK if structure exists but confirmations are insufficient
- INVALID if flow and structure conflict strongly
